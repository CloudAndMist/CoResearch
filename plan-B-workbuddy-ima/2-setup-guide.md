# Plan B：纯 IMA + WorkBuddy 研究组知识库部署指南

> **适用场景**：追求「零配置、开箱即用」的研究组  
> **核心工具**：IMA（知识库）+ WorkBuddy（自动化）  
> **部署时间**：约 20 分钟（管理员）+ 5 分钟/成员

---

## 一、方案概述

### 1.1 架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                     本地层（每位成员）                            │
│   IMA 客户端（Windows/Mac）                                       │
│   ├── 个人知识库（本地存储，30GB免费）                            │
│   ├── 共享知识库（团队协作）                                      │
│   └── 支持 PDF/网页/笔记/图片                                    │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 手动/半自动同步
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     自动化层（管理员 WorkBuddy）                  │
│   每周六 16:30：Lint 检查（质量报告）                              │
│   每周六 17:00：生成组会议题                                      │
│   └── 保存到 IMA 共享知识库「组会讨论」文件夹                        │
└───────────────────────────┬─────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     通知层（可选）                                │
│   WorkBuddy 接入微信 → 定时通知成员「议题已生成」                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 各角色职责

| 角色 | 任务 | 耗时 |
|------|------|------|
| **管理员** | 配置 IMA 共享库、设置 WorkBuddy 自动化、（可选）接入微信通知 | 20 分钟 |
| **成员** | 安装 IMA、加入共享库、日常使用 | 5 分钟 |

## 二、管理员部署步骤

### 2.1 配置 IMA 共享知识库（10分钟）

#### 步骤 1：创建共享知识库

1. 打开 IMA 客户端
2. 点击左侧「共享知识库」→ 新建知识库
3. 命名：`XX研究组共享库`
4. 设置权限：
   - 管理员：可编辑
   - 成员：可编辑（或仅查看，根据需要）

#### 步骤 2：创建「组会讨论」文件夹

1. 进入共享知识库
2. 新建文件夹：`📁 03-组会讨论`
3. 此文件夹用于存放自动生成的周会议题

#### 步骤 3：邀请成员

1. 进入共享知识库 → 点击「邀请成员」
2. 复制邀请链接
3. 将链接分发给所有成员（可通过微信/邮件）

#### 步骤 4：获取 IMA API Key

1. 打开 IMA → 点击左下角头像 → 设置
2. 找到「IMA Skills」或「开发者选项」
3. 点击「获取 API Key」
4. **保存 API Key**（仅显示一次！）

详细 API 使用说明见：[6-ima-skills-api-guide.md](./6-ima-skills-api-guide.md)

---

### 2.2 配置 WorkBuddy 自动化（10分钟）

#### 自动化 1：Lint 检查（每周六 16:30）

1. 打开 WorkBuddy → 自动化 → 新建自动化
2. 配置如下：

```toml
name = "研究组知识库 Lint 检查"
schedule = "每周六 16:30"
status = "active"

prompt = """
请执行研究组知识库 Lint（质量检查）任务：

## 配置信息
IMA_API_KEY = "{{IMA_API_KEY}}"
SHARED_KB_ID = "{{SHARED_KB_ID}}"

## 任务步骤

### 步骤 1：获取本周及上周内容
调用 IMA API 获取过去 14 天的所有条目：
- 时间范围：{{last_saturday}} 至 {{today}}
- 用于检测重复内容

### 步骤 2：执行 Lint 检查

#### 检查项 1：重复内容检测
- 比较标题相似度（>80% 视为重复）
- 比较内容摘要相似度（>70% 视为重复）
- 记录重复条目 ID 和标题

#### 检查项 2：信息完整性检查
- 标题是否清晰（长度 5-100 字）
- 是否有内容摘要（非空检查）
- 是否有来源信息（论文链接、会议名称等）
- 是否有标签/分类

#### 检查项 3：标签规范性检查
- 标签是否统一（如 "Transformer" vs "transformer"）

### 步骤 3：生成 Lint 报告

报告格式：
```
🔍 知识库 Lint 报告（{{date}}）

## 📊 总体统计
- 本周新增：X 条
- 总条目数：Y 条
- 健康度评分：Z/100

## ⚠️ 发现的问题
（列出重复、缺失、标签问题）

## 🎯 建议行动
（给出具体优化建议）
```

### 步骤 4：输出结果
- 在 WorkBuddy 结果面板展示 Lint 报告
- 如果健康度 < 70，标记为 🔴 需要关注
- 如果健康度 70-90，标记为 🟡 建议优化
- 如果健康度 > 90，标记为 🟢 状态良好
"""
```

#### 自动化 2：生成组会议题（每周六 17:00）

1. 再次新建自动化
2. 配置如下：

```toml
name = "研究组周会总结"
schedule = "每周六 17:00"
status = "active"

prompt = """
请执行研究组周会总结任务：

## 配置信息
IMA_API_KEY = "{{IMA_API_KEY}}"
SHARED_KB_ID = "{{SHARED_KB_ID}}"
LLM_API_KEY = "{{LLM_API_KEY}}"  # 可选，用于生成议题

## 任务步骤

### 步骤 1：获取本周新增内容
调用 IMA API 获取共享知识库本周新增的笔记/文档：
- 时间范围：上周六至今
- API Key: {{IMA_API_KEY}}

### 步骤 2：生成组会议题
- 使用 LLM 分析本周内容
- 生成 3-5 个讨论议题
- 每个议题包含：
  * 标题
  * 背景（1-2句话）
  * 讨论点（2-3个）
  * 相关文献链接

### 步骤 3：保存到 IMA 共享知识库
创建新笔记：
- 位置：共享知识库/📁 03-组会讨论/
- 标题：[{{date}}] 周会议题
- 标签：#会议纪要 #自动化生成
- 内容：Lint 报告摘要 + 组会议题

## 输出格式示例

# [{{date}}] 周会议题

> 自动生成时间：{{datetime}}
> 本周新增内容：X 条

---

## 🔍 Lint 报告摘要

- 健康度评分：85/100 🟡
- 发现问题：重复内容 1 组，信息缺失 2 条
- 建议：合并重复笔记，补充缺失信息

---

## 📝 组会议题

### 议题 1：[主题名称]
**背景**：简述这个议题的来源和重要性

**讨论点**：
- 问题 1
- 问题 2
- 问题 3

**相关文献**：
- [论文标题](IMA内链接)

---

### 议题 2：[主题名称]
...

---

## 💡 备注

- 本议题由 WorkBuddy 自动生成
- 如有问题请联系管理员
"""
```

### 2.3 （可选）接入微信通知

如果你希望成员在周六收到「议题已生成」的提醒，可以配置微信通知：

#### 方案 A：WorkBuddy 微信集成

1. 在 WorkBuddy 设置中绑定微信
2. 在自动化配置中添加微信通知步骤：

```toml
# 在周会总结自动化末尾添加

### 步骤 4：微信通知成员
发送微信消息到研究组群：

"""
📢 本周组会议题已生成

时间：{{date}}
新增内容：X 条
生成议题：Y 个

👉 请打开 IMA 共享知识库查看
路径：共享库 → 📁 03-组会讨论 → [{{date}}] 周会议题

周六组会时一起讨论！
"""
```

#### 方案 B：手动通知

如果 WorkBuddy 暂不支持微信，可以：
1. 周六 17:00 后手动查看 WorkBuddy 自动化结果
2. 复制生成的议题摘要
3. 发送到研究组微信群

## 三、成员使用指南

### 3.1 安装 IMA（5分钟）

#### Windows 用户
1. 访问 https://ima.qq.com
2. 点击「下载 Windows 版」
3. 安装并登录（支持微信扫码登录）

#### Mac 用户
1. 访问 https://ima.qq.com
2. 点击「下载 Mac 版」
3. 安装并登录

### 3.2 加入共享知识库

1. 打开 IMA 客户端
2. 点击管理员发送的邀请链接
3. 或：点击「共享知识库」→「加入知识库」→ 输入邀请码

### 3.3 日常使用流程

```
日常研究 → 保存到 IMA → 整理笔记 → 标记共享 → 周六组会讨论
```

#### 添加内容到知识库

| 内容类型 | 操作方式 |
|----------|----------|
| PDF 论文 | 拖拽到 IMA 窗口，或点击「上传文件」 |
| 网页/推文 | 复制链接 → 在 IMA 中粘贴 → 自动解析 |
| 微信文章 | 点击右上角「···」→ 分享到 IMA |
| 手写笔记 | 直接在 IMA 中新建笔记 |
| 截图/图片 | 截图后粘贴到 IMA 笔记中 |

#### 使用 AI 问答
1. 打开任意知识库
2. 点击底部输入框
3. 输入问题，如：「这篇论文的主要贡献是什么？」
4. IMA 会基于知识库内容回答

## 四、每周工作流程

### 4.1 成员端（每周六前）

```
周一 ~ 周六：
├── 阅读论文 → 保存到个人知识库
├── 做笔记、标注重点
└── 将需要分享的笔记「添加到共享知识库」

周六 17:00 前：
└── 确认本周想讨论的内容已在共享库中
```

### 4.2 自动化端（每周六）

```
周六 16:30：
├── WorkBuddy 自动执行 Lint 检查
├── 生成质量报告
└── 在 WorkBuddy 结果面板展示（仅管理员可见）

周六 17:00：
├── WorkBuddy 自动生成组会议题
├── 保存到 IMA 共享库「📁 03-组会讨论」
└── （可选）微信通知成员
```

### 4.3 组会时使用

1. 打开 IMA 共享知识库
2. 进入「📁 03-组会讨论」文件夹
3. 打开最新的「[日期] 周会议题」笔记
4. 基于议题进行讨论
5. 讨论后可在 IMA 中记录会议纪要

## 六、常见问题 FAQ

### Q1: IMA 免费版有什么限制？
- 个人知识库：30GB 存储空间
- 共享知识库：无明确限制，适合小型研究组
- 每日 AI 问答次数：有限制（具体以官方为准）

### Q2: 为什么去掉飞书？
- 飞书仅用于消息推送，而组会时本来就会一起查看议题
- 去掉飞书可以简化工具链，降低配置复杂度
- 如果确实需要通知，可以使用 WorkBuddy 微信集成

### Q3: 成员如何知道议题已生成？
- **推荐**：组会时一起查看 IMA 共享库
- **可选**：管理员配置 WorkBuddy 微信通知

### Q4: 如何导出知识库内容？
- IMA 支持导出笔记为 Markdown/PDF
- 共享知识库内容可以随时下载备份

### Q5: 离线时可以使用吗？
- IMA 本地知识库支持离线查看
- 共享知识库需要联网同步

### Q6: 如果 WorkBuddy 自动化失败怎么办？
- 检查 WorkBuddy 运行日志
- 确认 IMA API Key 有效
- 手动执行一次测试
- 如仍失败，可手动整理本周议题

## 七、进阶配置（可选）

### 7.1 自定义自动化 Prompt

如果需要更精细的议题生成，可以自定义 Prompt：

```
你是一位研究组助手。请基于本周知识库新增内容，生成组会议题。

要求：
1. 识别出 3-5 个值得讨论的主题
2. 每个主题包含：
   - 主题名称
   - 背景简述（1-2句话）
   - 具体讨论点（2-3个）
   - 相关文献/笔记链接
3. 标注出需要重点讨论的内容
4. 格式为 Markdown

本周新增内容：
{{ content }}
```

### 7.2 集成其他 LLM

如果 IMA 内置 AI 不满足需求，可以：
1. 在自动化中调用 Claude/GPT/DeepSeek API
2. 将 IMA 内容作为上下文传入
3. 获取更专业的分析结果

详细配置见：[5-automation-config.md](./5-automation-config.md)
