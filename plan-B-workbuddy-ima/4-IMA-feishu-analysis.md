# Plan B 可行性分析报告：IMA vs 飞书 独立方案

> **分析日期**：2026-04-14  
> **分析目标**：判断是否可以仅用 IMA 或仅用飞书实现「本地+云端协同知识库 + 每周自动总结」的完整范式

---

## 一、关键结论（TL;DR）

| 方案 | 可行性 | 结论 |
|------|--------|------|
| **仅用 IMA** | ⚠️ 部分可行 | 支持本地+云端知识库，但**缺少定时自动化能力**，无法实现「每周日17:00自动总结」 |
| **仅用飞书** | ⚠️ 部分可行 | 支持云端知识库+定时自动化，但**缺少本地个人知识库层**，成员无法本地管理 |
| **IMA + 飞书组合** | ✅ 推荐 | IMA 负责个人本地知识库，飞书负责团队云端+定时自动化，两者互补 |

**最终建议**：采用 **IMA（本地个人层）+ 飞书（云端团队层+定时自动化）** 的组合方案，这是目前最完整且可行的架构。

---

## 二、IMA 能力边界分析

### 2.1 IMA 已确认的能力 ✅

| 能力 | 支持情况 | 说明 |
|------|----------|------|
| **本地知识库** | ✅ 完整支持 | 个人知识库（30GB免费空间） |
| **共享知识库** | ✅ 完整支持 | 可创建多个共享库，邀请成员协作 |
| **API 接入** | ✅ 支持 | IMA Skills 提供 API Key，支持：读取知识库列表、搜索查询、写入网页/笔记、上传文件 |
| **多格式支持** | ✅ 支持 | PDF、Word、图片、网页、微信文章、文本笔记 |
| **LLM 问答** | ✅ 内置 | 基于混元/DeepSeek，支持 RAG 问答 |
| **跨端同步** | ✅ 支持 | Windows/Mac/小程序/网页版 |

### 2.2 IMA 缺失的能力 ❌

| 能力 | 支持情况 | 说明 |
|------|----------|------|
| **定时任务/自动化** | ❌ 不支持 | IMA Skills 仅提供 API 调用能力，**没有内置的定时调度机制** |
| **组会议题生成** | ❌ 不支持 | 需要外部 LLM 调用和 Prompt 工程 |
| **消息推送（群通知）** | ❌ 不支持 | 无法直接向微信群/飞书群推送消息 |
| **批量导出/同步** | ⚠️ 有限支持 | 可手动导出，无自动化同步机制 |

### 2.3 IMA API 详情

```yaml
# IMA Skills API 能力
获取地址: https://ima.qq.com/agent-interface
支持操作:
  - 知识库列表读取
  - 知识库搜索查询
  - 将网页加入知识库
  - 上传文件到知识库
  - 新建/追加文本笔记

限制:
  - 需要手动获取 API Key（仅显示一次）
  - 无定时调度能力
  - 无消息推送能力
```

## 三、飞书能力边界分析

### 3.1 飞书已确认的能力 ✅

| 能力 | 支持情况 | 说明 |
|------|----------|------|
| **云端知识库** | ✅ 完整支持 | 知识库空间，树形目录结构 |
| **API 接入** | ✅ 完整支持 | 飞书开放平台，200+ API 接口 |
| **定时自动化** | ✅ 支持 | 飞书多维表格自动化 + 飞书 CLI + GitHub Actions |
| **消息推送** | ✅ 支持 | 飞书机器人 Webhook，可推送至群组 |
| **权限管理** | ✅ 支持 | 细粒度的文档/知识库权限控制 |
| **飞书 CLI** | ✅ 支持 | 官方命令行工具，支持文档/知识库操作 |

### 3.2 飞书缺失的能力 ❌

| 能力 | 支持情况 | 说明 |
|------|----------|------|
| **本地个人知识库** | ❌ 不支持 | 飞书是纯云端产品，**没有本地客户端存储** |
| **离线使用** | ❌ 不支持 | 必须联网使用 |
| **个人免费版限制** | ⚠️ 有限制 | 个人版知识库空间仅 2GB，企业版才适合研究组 |

### 3.3 飞书 CLI 能力

```yaml
# 飞书 CLI 功能
安装: npm install -g feishu-cli
支持:
  - 文档创建/读取/更新
  - 知识库节点管理
  - 文件上传/下载
  - 消息发送
  - 任务管理

不支持:
  - 定时任务调度（需结合系统 cron 或 GitHub Actions）
  - 本地文件夹自动同步
```

## 五、单元测试结果：IMA 本地访问

### 测试环境
- **IMA 版本**：ima.copilot（Windows 桌面版）
- **运行状态**：✅ 已运行（进程 ID: 600, 7340, 9844...）
- **本地 API 端口**：❌ 未开放（测试 8080 端口失败）

### 测试结论

```
IMA 本地进程运行正常，但未开放本地 HTTP API 端口。
这意味着：外部程序无法直接通过 localhost 调用 IMA。

可行的访问方式：
1. IMA Skills API Key（云端 API）
2. 用户手动在 IMA 客户端中操作

不可行的访问方式：
- 直接读取本地 IMA 数据库/文件
- 通过本地端口调用 IMA 功能
```

## 六、最终方案建议

### 推荐架构：IMA + 飞书 组合

```
┌─────────────────────────────────────────────────────────────────┐
│                     本地层（每位成员）                            │
│   IMA 客户端（Windows/Mac）                                       │
│   ├── 个人知识库（本地存储，30GB）                                │
│   ├── 共享知识库（团队协作）                                      │
│   └── 支持 PDF/网页/笔记/图片                                    │
└───────────────────────────┬─────────────────────────────────────┘
                            │ 手动/半自动同步
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     自动化层（每周日17:00触发）                   │
│   WorkBuddy 自动化 / GitHub Actions / 飞书多维表格自动化          │
│   ├── 调用 IMA API 获取本周新增内容                              │
│   ├── 调用 LLM 生成组会议题                                      │
│   └── 推送至飞书群组                                             │
└───────────────────────────┬─────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     云端层（团队共享）                            │
│   飞书知识库（可选，用于存档）                                    │
│   飞书群组（接收每周议题推送）                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 成员使用流程

```
1. 安装 IMA 客户端（5分钟）
2. 创建个人知识库，设置共享知识库
3. 日常：在 IMA 中阅读论文、整理笔记
4. 每周日前：将需要分享的笔记标记为「共享」
5. 每周日17:00：自动收到飞书群的组会议题
```
